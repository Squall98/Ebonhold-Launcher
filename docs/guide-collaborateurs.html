Ajouter un addon au launcher

Guide pour l'équipe — ajouter ou mettre à jour un mod dans le catalogue, sans toucher au code.

Le principe en une phrase. Chaque addon a son propre dépôt GitHub. Le catalogue du launcher se construit tout seul à partir des Releases de ces dépôts — tu n'édites jamais de fichier compliqué et tu ne calcules jamais de code de vérification.

1. Mettre à jour un addon existant

C'est le cas le plus courant, et c'est le plus simple : il n'y a rien à faire sur le launcher.

  1. Sur le dépôt GitHub de l'addon, crée une nouvelle Release (ex. v1.3) avec le fichier .zip de l'addon en pièce jointe.
  2. C'est tout. Le launcher détecte la nouvelle version automatiquement et la propose aux joueurs (dans les deux heures, ou tout de suite si un admin relance la mise à jour).

Astuce : si une version n'est pas prête, coche « pre-release » sur la Release — elle sera ignorée tant qu'elle reste en pre-release.

2. Ajouter un nouvel addon

Deux méthodes au choix. Aucune ne demande de connaissances techniques particulières.

Méthode A · recommandée

Le formulaire

Tu remplis un formulaire sur GitHub. Un robot vérifie tout et prépare l'ajout à ta place.

Méthode B

Le petit fichier

Tu crées un mini-fichier de quelques lignes via l'éditeur GitHub. Pour les habitués.

Méthode A — Le formulaire

  1. Ouvre la page de soumission :
    Ouvrir le formulaire « Ajouter un addon »
    (ou : onglet Issues du dépôt → New issue → « Ajouter un addon au launcher »)
  2. Remplis les champs (voir la maquette ci-dessous), puis clique sur Submit new issue.
  3. Un robot vérifie le dépôt et crée une demande d'ajout (Pull Request) tout seul, puis te répond dans l'issue avec le lien.
  4. Un admin relit, valide, et l'addon apparaît dans le launcher. Terminé.

Méthode B — Le petit fichier

  1. Dans le dépôt, va dans le dossier catalog/ et ouvre _TEMPLATE.toml pour t'en inspirer.
  2. Clique Add file → Create new file, nomme-le catalog/mon-addon.toml et recopie le modèle.
  3. Remplis les champs (voir le tableau ci-dessous), puis Propose changes : ça crée automatiquement la demande d'ajout.
  4. Un admin la valide. C'est en ligne.

La version, le lien de téléchargement, le code de vérification et l'historique ne se renseignent pas : ils viennent automatiquement de la dernière Release du dépôt.

Questions fréquentes

Il faut quoi sur le dépôt du mod pour qu'il marche ?

Au moins une Release GitHub avec un fichier .zip attaché (ou .mpq pour un patch de données). C'est ce zip que le launcher télécharge et installe.

Comment je publie une mise à jour plus tard ?

Tu n'as rien à changer dans le catalogue : tu crées simplement une nouvelle Release sur le dépôt du mod. Le launcher prend la dernière version automatiquement.

Le texte « Nouveautés » vient d'où ?

De la description de ta Release GitHub (sa première phrase). Donc soigne un peu tes notes de Release : elles s'affichent telles quelles dans le launcher.

Est-ce que mon ajout est publié tout de suite ?

Non. Un ajout passe d'abord par une demande (Pull Request) qu'un admin relit et valide. C'est une sécurité : rien n'arrive aux joueurs sans relecture.

J'ai fait une erreur dans le formulaire, c'est grave ?

Non. Si quelque chose ne va pas, le robot te le dit directement dans ton issue. Tu corriges, il réessaie. Rien n'est cassé entre-temps.
